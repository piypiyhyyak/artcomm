Этот файл нужен, чтобы не вспоминать каждый раз, **куда класть медиа** и **как правильно их подключать**.

## Главное: куда загружать файлы

Если вам прислали фото, видео, логотипы и нужно быстро вставить на сайт:

- кладите изображения и SVG в `public/assets`
- видео `*.mp4` в Git не добавляйте (правило `*.mp4` в `.gitignore`)
- для продакшена загружайте mp4 напрямую на сервер, например в `/var/www/html/assets/`

Пример:

- `public/assets/hero-1.jpeg`
- `public/assets/expert-1.png`
- `public/assets/logo-mark.png`

---

## Чем отличается `public/assets` от `src/assets`

### `public/assets` — статика “как есть”

Это папка для файлов, которые браузер должен получать напрямую по URL.

Пример:

- файл: `public/assets/logo-mark.png`
- путь на сайте: `/assets/logo-mark.png`

Где удобно использовать:

- в HTML-строках
- в `background-image: url('/assets/...')`
- в `<img src="/assets/...">`

Плюс: просто и предсказуемо.  
Минус: Vite не оптимизирует эти файлы автоматически.

---

### `src/assets` — файлы внутри кода React

Это вариант, когда файл импортируется в JS/TS:

```jsx
import logo from './assets/logo.png';
```

Тогда сборщик сам обработает файл, переименует и положит в `dist`.

Плюс: удобно для компонентов и контроля через импорты.  
Минус: нельзя просто ссылаться строкой `"/assets/..."`, нужен импорт.

## Что использовать в этом проекте сейчас

С текущей структурой проекта и вашим процессом лучше всего:

- **изображения хранить в `public/assets`, а mp4 держать вне репозитория**
- подключать видео публичным путём: `/assets/gimn-ed-zy9mar.mp4`

Так проще менять контент без лишней возни.

## Безопасность CMS (прод)

Логин/пароль администратора и кодовое слово CMS должны храниться **в окружении сервера**, не в репозитории.

1. Скопируйте шаблон:

```bash
sudo install -m 600 -o root -g root deploy/systemd/artcomm-cms.env.example /etc/artcomm-cms.env
```

2. Отредактируйте `/etc/artcomm-cms.env` и задайте реальные секреты:

- `ARTCOMM_CMS_ADMIN_LOGIN`
- `ARTCOMM_CMS_ADMIN_PASSWORD`
- `ARTCOMM_CMS_SECURITY_CODEWORD`

3. Перезапустите сервис CMS:

```bash
sudo systemctl daemon-reload
sudo systemctl restart artcomm-cms
```

## Частые проблемы (и как быстро починить)

### 1) “Я заменил файл, но на сайте старое изображение”

Сделайте жесткое обновление страницы:

- Mac: `Cmd + Shift + R`

### 2) “Путь есть, но картинка не грузится”

Проверьте:

- файл реально доступен на сервере по URL `/assets/...`
- в коде путь начинается с `/assets/...`
- нет лишних пробелов/опечаток в имени

### 3) “Слишком длинные/сложные имена файлов”

Лучше делать короткие понятные имена:

- `hero-1.jpeg`, `hero-2.jpeg`
- `expert-1.png`
- `logo-mark.png`

Так меньше шансов словить баги с путями.
